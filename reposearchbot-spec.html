📋 RepoResearchBot Spec

OpenClaw instance for Discord - GitHub Repo Research

🏗️ Architecture

graph TB subgraph DISCORD["💬 Discord #repo-research"] U[User] MSG[Message] end subgraph OPENCLAW["📦 OpenClaw Container"] G[Gateway] A[Agent] GITHUB[GitHub API] WEB[Web Search] end U --> MSG --> G G --> A A --> GITHUB A --> WEB A --> MSG style DISCORD fill:#1f2937,color:#fff style OPENCLAW fill:#1e1b4b,color:#fff style G fill:#4c1d95,color:#fff style A fill:#4c1d95,color:#fff end

🎯 What It Does

Core Functions
1. Reads spec/idea from Discord channel
2. Searches GitHub for relevant repos
3. Returns clickable links (formatted for Discord)
4. Summarizes each repo (stars, forks, description)
5. Later: clone, analyze, customize repos

👤 User Flow

sequenceDiagram participant User participant Discord participant OpenClaw participant GitHub User->>Discord: "I need a kanban board for marketing" Discord->>OpenClaw: Message received OpenClaw->>GitHub: Search repos GitHub-->>OpenClaw: [repo1, repo2, repo3] OpenClaw->>OpenClaw: Format for Discord OpenClaw->>Discord: 📋 Results: Discord-->>User: Links + summaries end

🛠️ Tech Stack

• OpenClaw (the gateway)
• Discord channel as input
• GitHub API for search
• Web search for research
• MiniMax/Grok for understanding

💬 Discord Output Format

graph LR OUT[Output] --> L1[📋 Title] OUT --> L2[⭐ Stars | 🍴 Forks] OUT --> L3[📝 Description] OUT --> L4[🔗 github.com/...] style OUT fill:#1e3a8a,color:#fff end
