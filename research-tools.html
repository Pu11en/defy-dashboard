🔬 Architecture Research Tools

OSS tools to research & map complex architectures

📊 Code Analysis & Mapping
Swark (VS Code)
github.com/swark-io/swark
AI generates architecture diagrams from code using GitHub Copilot. Analyzes your codebase and draws the structure.
→ Use: Analyze any repo → get diagram
CodeScene
github.com/codescene-io
Behavioral code analysis. Finds hotspots, complexity trends, and social patterns in code.
→ Use: Understand code complexity + ownership
sonarqube
github.com/SonarSource/sonarqube
Code quality & security. Continuously analyze code to find bugs, vulnerabilities, tech debt.
→ Use: Audit code quality
📐 Diagramming & Visualization
Mermaid.js
github.com/mermaid-js/mermaid
Text-to-diagram. Write diagram code, renders as SVG. GitHub native. Flowcharts, sequence, class, state, ER, Gantt.
→ Use: Embed diagrams in docs (I use this!)
Graphviz
github.com/graphviz
Open source graph visualization. Converts description language to images. Industry standard.
→ Use: Complex dependency graphs
PlantUML
github.com/plantuml/plantuml
Text to UML diagram. All UML types, network, JSON, YAML visualizations. Very powerful.
→ Use: Full UML diagrams
📚 Learning Resources
awesome-software-architecture
github.com/mehdihadeli/awesome-software-architecture
Curated list of articles, videos, resources to learn software architecture, patterns, principles.
→ Use: Learn architecture best practices
awesome-system-design
github.com/madd86/awesome-system-design
Curated resources for System Design (Distributed Systems). Theory + practice.
→ Use: Learn distributed systems
Architecture of Open Source Apps
github.com/kimtth/aosa_pdf
Free PDF books analyzing architecture of real open source apps. Volume 1 & 2.
→ Use: See how real apps are architected
🤖 AI-Powered
Eraser (DiagramGPT)
eraser.io
AI generates diagrams from text. Describe what you want → it draws. Cloud architecture, flowcharts, ERD.
→ Use: "Draw me a system that does X"
Structurizr
github.com/structurizr
C4 model for architecture. Create multiple views of your system (context, container, component, code).
→ Use: Professional architecture docs
