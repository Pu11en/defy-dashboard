🚀 Scaling OpenClaw - Backend Options

Multiple instances on your machine

1. Docker Compose (Recommended)
Run multiple isolated OpenClaw containers. Each gets own port, config, memory.
docker run -d --name openclaw-agent1 -p 18801:3001 -e OPENCLAW_GATEWAY_TOKEN=token1 openclaw/openclaw:latest
Isolated Portable Easy to manage
2. Multiple Profiles
Run multiple OpenClaw with different config files on same machine.
openclaw --profile agent1 gateway openclaw --profile agent2 gateway
No Docker Lightweight Same machine
3. Sub-agents (What we have!)
Spawn parallel agents from one OpenClaw. Each gets own context + model.
sessions_spawn(model="grok-4.1", task="work on X")
Already working Shared memory Fast spawn
4. Kubernetes (Advanced)
Run at scale with multiple machines. Auto-scaling, load balancing.
